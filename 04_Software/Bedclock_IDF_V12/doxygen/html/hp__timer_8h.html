Detailed Description

microsecond-precision 64-bit timer API

Can be used for single shot timers or for looping (auto reset) timers Beware that an auto reset timer will only be reset if and when hp_timer_has_lapsed() or hp_timer_lapsed() are called.

If auto_reset is enabled, the interval is added to the start time multiple times until the next trigger moment is in the future.

This means that if intervals have been missed because hp_timer_has_lapsed() or hp_timer_lapsed() were not called frequently enough, trigger signals will not accumulate and clog the system.

Just for fun: 

 esp_timer_get_time() - timer->last_reset_time) < timer->interval

is used to ensure that the timer also works if the clock passes the 64 bit mark, (250.000 years after switching on the device).
